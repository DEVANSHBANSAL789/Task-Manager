# Team Task Manager

A full-stack collaborative application where multiple users can manage tasks efficiently. Think of it as a simplified version of tools like Trello or Asana.

## Features
- **User Authentication:** Signup and login using JWT.
- **Project Management:** Create projects (creator becomes Admin). Admin can add/remove members. Members can view assigned projects.
- **Task Management:** Create tasks (Title, Description, Due Date, Priority), assign tasks to users, and update status (To Do, In Progress, Done).
- **Dashboard:** Overview of total tasks, tasks by status, tasks per user, and overdue tasks.
- **Role-Based Access:** Admins manage tasks and users. Members view and update assigned tasks only.
- **Responsive Design:** Premium dark UI Custom CSS.

## Tech Stack
- **Frontend:** React.js (Vite), React Router, Axios, Custom CSS.
- **Backend:** Node.js, Express.js.
- **Database:** MongoDB (Mongoose).

## Local Development Setup

### Prerequisites
- Node.js installed
- MongoDB Atlas URI

### 1. Clone the repository
```bash
git clone <your-repo-url>
cd task-manager
```

### 2. Backend Setup
```bash
cd backend
npm install
```
Create a `.env` file in the `/backend` directory:
```env
PORT=5000
MONGO_URI=mongodb://localhost:27017/taskmanager
JWT_SECRET=your_super_secret_jwt_key
```
Start the backend server:
```bash
node server.js
```

### 3. Frontend Setup
Open a new terminal and run:
```bash
cd frontend
npm install
npm run dev
```
The frontend will start on `http://localhost:5173`.

## Deployment to Railway (Mandatory Step)

To deploy the full-stack application to Railway, we will deploy the backend and frontend as two separate services within the same Railway project.

### Part 1: Deploy Backend
1. Go to [Railway.app](https://railway.app/) and create a new project.
2. Select **"Provision MongoDB"** to create a cloud database.
3. Once MongoDB is provisioned, go back to the Project Dashboard.
4. Click **"New" -> "GitHub Repo"** and select your repository.
5. Railway will detect the monorepo structure. To deploy the backend:
   - Go to **Settings -> Build -> Root Directory** and set it to `/backend`.
6. Go to **Variables** and add:
   - `MONGO_URI` (Use the connection string from the provisioned Railway MongoDB).
   - `JWT_SECRET` (e.g., `my_secret_key_123`).
   - `PORT` (Railway sets this automatically, but you can set `5000`).
7. Wait for the deployment to finish, and copy the **Public Domain** generated by Railway (e.g., `https://backend-production-xyz.up.railway.app`).

### Part 2: Deploy Frontend
1. In the same Railway project, click **"New" -> "GitHub Repo"** and select the *same* repository again.
2. For this new service, go to **Settings -> Build -> Root Directory** and set it to `/frontend`.
3. In the `/frontend/src/context/api.js` file, ensure the `baseURL` points to your newly deployed backend URL instead of `localhost`. *Alternatively, you can use Vite environment variables (e.g., `VITE_API_URL`) and set them in Railway.*
4. Add the start command for Vite in the Railway settings if needed, or let Railway detect it.
5. Generate a **Public Domain** for the frontend service.

Your application is now live!
